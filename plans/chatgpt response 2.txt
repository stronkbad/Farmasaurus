## 1. Ultima Online (original design)

**Ultima Online**

**Facts (documented):**

* Server-authoritative
* Tile-based movement
* Discrete step validation
* Client animates steps before server confirmation
* Server rejects illegal steps instead of correcting continuous motion

**Primary sources**

* Raph Koster (lead designer) has repeatedly described UO movement as **tile-step validated**, not continuous.
* UO movement packets were *directional commands*, not position streams.

**Why this matters**

> UO avoided rubberbanding not because of low precision, but because **the server validated intent before committing state**.

---

## 2. Tibia (still running today)

**Tibia**

**Facts:**

* Entirely tile-based to this day
* Server validates every step
* Client interpolates visuals
* Rubberbanding is almost nonexistent even on high latency

**Evidence**

* Open Tibia server implementations (`canWalk()`, `canStep()`) expose explicit movement validators.
* Tibia allows massive worlds (hundreds of millions of tiles).

**Conclusion**

> Tile-based validation scales better and feels *more stable* than continuous correction.

---

## 3. Albion Online (modern MMO)

**Albion Online**

**Facts:**

* Isometric
* Server authoritative
* Uses **micro-tiles** (still discrete)
* Client interpolates heavily
* No continuous steering authority

Albion explicitly moved *away* from continuous client steering early in development due to jitter.

## 4. Industry-standard networking reference

**Game Programming Gems**
**Gaffer on Games**

Specifically:

* *“Client-side prediction with server reconciliation”*
* *“Deterministic lockstep vs authoritative server models”*

**Key quote (paraphrased, Glenn Fiedler):**

> If the server corrects the client every frame, the system is unstable. Corrections must be discrete and bounded.

Your system currently violates this.

## 5. Control theory (not game-specific)

What you built is a **closed feedback loop**:

* Output (movement) feeds back into input (direction calc)
* With delay (network)
* Without damping (intent latching)

That is mathematically unstable.
This is not subjective; it is textbook feedback behavior.
